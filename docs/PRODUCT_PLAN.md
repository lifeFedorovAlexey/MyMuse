# Product Plan: MyMuse

## 1. Видение

MyMuse - кроссплатформенный self-hosted музыкальный сервис с современным UX в духе Spotify / YouTube Music / Яндекс Музыки, где:

- каждый пользователь хранит музыку на своей инфраструктуре;
- владелец инстанса управляет доступом и шарингом;
- сервис не является поставщиком контента и не отвечает за правовой статус загружаемых файлов.

## 2. Целевая аудитория

- пользователи с личной медиатекой;
- семьи/друзья для закрытого обмена музыкой;
- small communities с приватным музыкальным сервером.

## 3. Продуктовые принципы

- Self-host first.
- Privacy by default.
- Fast streaming over weak networks.
- Единый UX на desktop + mobile.
- Расширяемость (AI-подборки, импорт из внешних источников).

## 4. Основные сценарии MVP

1. Установить сервер на свое устройство.
2. Загрузить локальные треки.
3. Прослушать трек онлайн.
4. Создать плейлист.
5. Поделиться ссылкой на трек/плейлист.
6. Подключиться к чужому инстансу по ссылке-приглашению.

## 5. Этапы реализации

## Этап 0: Foundation

- Репозиторий, архитектурные решения, policy.
- Базовый backend API и healthcheck.
- Конфигурация окружения (ENV, storage adapters).

## Этап 1: MVP Core

- Аутентификация (owner + invited users).
- Каталог треков (CRUD + metadata parsing).
- Стриминг файлов (range requests).
- Плейлисты.
- Share links (временные/постоянные).
- Базовый клиент (поиск, библиотека, player).

## Этап 2: Distribution & UX

- Установщик сервера (Docker/desktop helper).
- Desktop/mobile клиент с единым дизайном.
- Offline cache для клиента.
- Синхронизация прогресса прослушивания.

## Этап 3: Smart Features

- Нейро-подборки из существующей библиотеки.
- Семантический поиск по локальной медиатеке.
- Автоматические миксы.

## Этап 4: External Import (опционально)

- Коннекторы внешних источников.
- Поиск и добавление треков в плейлист с учетом legal/region limits.
- Логирование источника и статуса лицензии (если применимо).

## 6. Нефункциональные требования

- P95 старт трека < 1.5 сек в локальной сети.
- Поддержка больших библиотек (100k+ треков).
- Безопасный шаринг (подписанные токены, сроки жизни).
- Логи и аудит действий.

## 7. Риски

- Правовые риски при нарушении авторских прав пользователями.
- Разница производительности на слабых домашних серверах.
- Сложность стабильного UX на всех платформах.

## 8. Критерии успеха MVP

- Server install <= 10 минут.
- Upload and play <= 2 минуты после установки.
- Share link работает между двумя независимыми пользователями.
